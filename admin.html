Database

This console reads and writes directly to the live Supabase project. There's no export step anymore — edits here are the real data.

Admin login is real now

Signing in here uses a real Supabase account (Authentication → Users in the Supabase Dashboard), not a shared passcode. Row-level security only allows treatment, special, hours and booking-cancellation writes from a logged-in admin — everything else (customers booking, buying vouchers) works without login by design.

Public booking & voucher flow

Customers booking or buying a voucher on the public site don't need to log in — those writes are allowed by specific, narrow database rules (e.g. anyone can create a booking, but only an unredeemed voucher can be marked redeemed). Admin edits to treatments, specials and hours are the ones that need real login.
